title: Space Souls

NevoFlux's browser organizes your tabs into **Spaces**. With **Space Souls**, each
Space can have its own assistant "soul" — a different persona, its own private memory,
and its own cookie jar — so a Work Space, a Personal Space, and a Research Space each
get an assistant tuned to that context.

## What a soul is

A soul is the agent's **persona** — the same Identity / Soul / User / Tools set you
shape in [Settings → General](/en/docs/settings/general#persona-soul-files). Space
Souls lets you assign a *different* one to each Space instead of one global persona.

## What's isolated per Space

A soul, its **private memory**, and the Space's **cookie jar** all share one key (the
Space's container). Switching Spaces therefore switches the assistant *and* what it
remembers *and* which logins it uses — cleanly separated, with no cross-contamination.

## Set it up

Bindings live in a `space_souls.toml` file that maps each Space to a soul. For the
binding format and the Space Souls settings, see
[Settings → General → Per-Space personas](/en/docs/settings/general#per-space-personas-space-souls).
A Space with no binding simply uses your single global persona, exactly as before.

## Why use it

*Example:* a **Research** Space whose soul knows your research context and keeps its
own notes; a **Work** Space with a professional persona and separate logins that never
mix with your personal browsing.

See also [Long-term memory](/en/docs/features/memory).

---
title: MCP Integration
description: Use external MCP tools inside NevoFlux, and expose NevoFlux to other MCP clients.
---

NevoFlux speaks the **Model Context Protocol (MCP)** in both directions.

## Connect external MCP servers

Add MCP servers (stdio / SSE / HTTP) in
[Settings → MCP Servers](/en/docs/settings/mcp-servers). Their tools become available to
the agent, which finds and calls the right one automatically (and can search them
with `tool_search`). NevoFlux supervises servers with auto-reconnect and health
checks; your list is stored at `~/.config/nevoflux/mcp-servers.toml`.

## Expose NevoFlux as an MCP server

The agent can also run **as an MCP server** so other MCP clients (such as Claude
Code) can use NevoFlux's browser and computer tools. Run the bundled agent binary in
MCP mode:

```bash
nevoflux --mcp
```

Then point your MCP client at it, e.g. for Claude Code:

```json
{
  "mcpServers": {
    "nevoflux": {
      "command": "/path/to/nevoflux",
      "args": ["--mcp"]
    }
  }
}
```

In this mode NevoFlux exposes its **browser** tools (navigate, click, type, fill,
screenshot, read content/markdown, snapshot, eval JS, …), its **computer** tools
(screenshot, mouse, keyboard, …), and an **`agent_chat`** tool.

## Tool permissions for CLI/agent providers

Some CLI/agent providers (notably [Antigravity](/en/docs/configuration/providers))
can't ask you for permission themselves. For those, NevoFlux enforces its normal
[permission prompts](/en/docs/features#permissions-human-in-the-loop) **server-side**
on every tool call the agent makes, so your Confirm-before-actions setting still
applies. Read-only tools (for example listing browser tabs) pass through without a
prompt.

> The agent binary is bundled inside the browser install; you can also build it from
> the [agent repo](https://github.com/nevoflux-browser/nevoflux-agent) for standalone use.

## 0.3.6

*发布于 2026-06-10*

本次发布的重点是 **Packs(扩展包)**—— 一种打包与分享 NevoFlux 能力的全新方式,并在
设置中提供一流的安装与管理体验,还支持直接从 GitHub 安装。

### 新增

- **Packs 设置面板。** 新增 **设置 → Packs** 区块,列出已安装的扩展包,并支持安装、
  更新、卸载与校验 —— 与知识库向导保持一致的体验。
- **从 GitHub 安装 Packs。** 除本地清单外,现在可以从远程 GitHub 源
  (`github:user/repo[/sub][@ref]` 或 `https://github.com/…` 链接)安装或更新扩展包。
  远程源会经过**检视 → 预览 → 确认**的流程:预览扩展包内容、标记其中携带的 canvas
  工具二进制文件、暴露会阻止安装的策略违规项、对未经审阅的来源给出警告,并要求你再次
  明确确认后才会真正安装。
- **扩展包开发参考。** 面向扩展包作者的完整指南,涵盖清单格式、所有组件类型、能力沙箱
  规则、生命周期与保证、通过 CLI 与 Packs 界面安装与管理、沙箱化测试、一个完整的示例
  以及常见问题解答。

### 修复

- **Windows 上 WebRTC 恢复可用。** 原生 Windows 构建重新启用了 WebRTC,恢复了
  `navigator.mediaDevices`。这修复了在缺少媒体设备 API 时登录流程卡住的站点(例如登录
  X/Twitter)。
- **不再出现启动时的 “Missing chrome or resource URL” 报错。** 现已正确注册分屏视图
  样式,并移除了一处内部重命名遗留的失效脚本引用。
- **扩展包安装对话框。** 成功后的自动关闭计时器现在会记录它所属的对话框,因此在短时间内
  再次打开安装对话框时,不会误关另一个对话框。

### 构建与 CI

- 加固了原生 Windows 发布流程:`mozmake` 的设置不再卡死,并且每个构建阶段都能用到
  `mozmake`,使 Windows 构建可以稳定链接。

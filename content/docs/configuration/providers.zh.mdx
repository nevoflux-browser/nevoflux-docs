NevoFlux 支持众多 LLM 提供商。在
[设置 → AI Models](/zh/docs/settings/ai-models) 中选择,或在
[`config.toml`](/zh/docs/configuration) 中设置 `provider`。

> **提供商类型。** AI Models 标签页将它们分为 **service**(云端 API)、
> **local**(Ollama)、**CLI**(`claude-code`、`antigravity`、`kimi-agent` —— 运行本地
> 命令行智能体,自行管理鉴权)和 **agent**(`openclaw`)。CLI/agent 类提供商以流式
> 输出;少数提供商(尤其是 **Ollama**)不支持流式。

> **Antigravity(v0.3.12 新增)。** 该提供商通过社区适配器
> [`antigravity-acp`](https://github.com/shubzkothekar/antigravity-acp) 驱动 Google 的
> `agy` CLI。它**免密钥** —— API 密钥留空即可,表中的 `GEMINI_API_KEY` 仅为占位符。
> 选用前请先安装两个命令行工具并加入 `PATH`:`agy` CLI 本身,以及 `antigravity-acp`
> 适配器二进制(从适配器 releases 下载后重命名为 `antigravity-acp`,Windows 上为
> `antigravity-acp.exe`)。模型保持 `default` 即可使用 agy 自身的默认模型。Antigravity
> 在 AI Models 标签页中取代了原来的 **Gemini CLI** 磁贴;若在 `config.toml` 中手动设置,
> `gemini-cli` 提供商仍然可用。

## 提供密钥

为提供商提供密钥有两种方式,智能体按此顺序查找:

1. **写入 `config.toml`** —— `[llm.<provider>]` 下的 `api_key`:

   ```toml
   [llm.anthropic]
   api_key = "sk-ant-..."
   ```

2. **环境变量** —— 上表中的变量,例如:

   ```bash
   export ANTHROPIC_API_KEY="sk-ant-..."
   ```

> NevoFlux **不会**把密钥存入操作系统钥匙串。请使用配置文件或环境变量。

## 自定义端点

在提供商下设置 `base_url`,指向代理或 OpenAI 兼容服务:

```toml
[llm.openai]
model = "my-model"
base_url = "https://my-gateway.example.com/v1"
```

## 切换当前模型

在[设置 → AI Models](/zh/docs/settings/ai-models)中切换当前提供商,编辑
`config.toml` 中的 `provider`,或在对话中让智能体切换模型(它具备 `switch_model`
能力)。

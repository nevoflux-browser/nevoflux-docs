## 0.3.6

*Released 2026-06-10*

The headline of this release is **Packs** — a new way to bundle and share NevoFlux
capabilities, with first-class install and management right inside Settings, including
installing straight from GitHub.

### New

- **Packs settings panel.** A new **Settings → Packs** section lists your installed packs
  and lets you install, update, uninstall, and validate them — mirroring the Knowledge
  Base wizard.
- **Install Packs from GitHub.** Install or update a pack from a remote GitHub source
  (`github:user/repo[/sub][@ref]` or a `https://github.com/…` URL) in addition to local
  manifests. Remote sources run an **inspect → preview → confirm** flow that previews the
  pack's contents, flags any bundled canvas-tool binaries, surfaces policy violations
  (which block the install), warns you about unreviewed sources, and requires an explicit
  second confirmation before anything is installed.
- **Pack development reference.** A comprehensive guide for pack authors covering the
  manifest format, every component type, the capability sandbox rules, lifecycle and
  guarantees, installing and managing via the CLI and the Packs UI, sandboxed testing, a
  complete worked example, and a FAQ.

### Fixed

- **WebRTC works again on Windows.** Native Windows builds ship with WebRTC enabled,
  restoring `navigator.mediaDevices`. This fixes sites whose login flow stalled when the
  media-device APIs were missing (for example, signing in to X/Twitter).
- **No more startup "Missing chrome or resource URL" errors.** Split-view styling is now
  registered correctly, and a stale script reference left over from an internal rename was
  removed.
- **Pack install dialog.** The success auto-close timer now tracks which dialog it belongs
  to, so quickly reopening the install dialog no longer closes the wrong one.

### Build & CI

- Hardened the native Windows release pipeline: the `mozmake` setup no longer hangs, and
  `mozmake` is now available to every build stage so Windows builds link reliably.
